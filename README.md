# Сравнение алгоритмов сортировки со слиянием

**Курсовая работа** по дисциплине "Структуры и алгоритмы обработки данных" (СИАКОД)

## 📋 Описание проекта

Этот проект реализует и анализирует четыре различных алгоритма сортировки со слиянием на языке **C++**:

1. **Двухфазная сортировка с простым слиянием** (Two-Phase Simple Merge Sort)
2. **Двухфазная сортировка с естественным слиянием** (Two-Phase Natural Merge Sort)
3. **Однофазная сортировка с простым слиянием** (One-Phase Simple Merge Sort)
4. **Однофазная сортировка с естественным слиянием** (One-Phase Natural Merge Sort)

Проект включает интерактивное консольное приложение для:
- Создания входных файлов с произвольным количеством чисел
- Выполнения сортировки различными методами
- Сбора статистики производительности алгоритмов

## 🎯 Основные возможности

### Функциональность

- ✅ Создание тестовых входных файлов с генерацией случайных чисел
- ✅ Реализация всех четырех вариантов алгоритма слияния
- ✅ Подсчет операций сравнения и перемещения элементов
- ✅ Измерение времени выполнения алгоритмов (в миллисекундах и наносекундах)
- ✅ Интерактивное меню на русском языке
- ✅ Обработка ошибок при работе с файлами

### Метрики производительности

Каждый алгоритм отслеживает следующие метрики:

```cpp
struct SortStats {
    long long cmp_cnt;      // Количество сравнений
    long long move_cnt;     // Количество перемещений
    double duration_ns;     // Время выполнения в наносекундах
};
```

## 🚀 Быстрый старт

### Требования

- **Компилятор**: MSVC (Visual Studio 2019 или выше) или любой C++17 совместимый компилятор
- **Стандарт C++**: C++17 или выше (используется `<filesystem>`)

### Компиляция и запуск

#### В Visual Studio

1. Откройте проект: `l5/l5.vcxproj`
2. Нажмите `Build > Build Solution`
3. Нажмите `Debug > Start Debugging` или `Ctrl+F5`

#### Командная строка (Windows)

```powershell
cd l5
cl /std:c++17 main.cpp /o main.exe
main.exe
```

#### Командная строка (Linux/macOS)

```bash
cd l5
g++ -std=c++17 main.cpp -o main
./main
```

## 🔍 Описание алгоритмов

### Двухфазная сортировка (Two-Phase)

**Распределение → Слияние**

1. **Фаза распределения**: Входной файл разбивается на два временных файла
2. **Фаза слияния**: Элементы из двух файлов сливаются обратно в исходный файл
3. **Повторение**: Процесс повторяется с удвоением размера серии на каждой итерации

**Особенности**:
- Требует дополнительное дисковое пространство (два временных файла)
- Хорошо подходит для внешней сортировки
- Полезна для сортировки файлов, которые не влезают в оперативную память

### Однофазная сортировка (One-Phase)

**Слияние в один проход**

1. Читаются серии из исходного файла
2. Серии сливаются прямо в выходной файл
3. На каждой итерации размер серии удваивается

**Особенности**:
- Требует только один входной файл
- Более эффективна по использованию дискового пространства
- Меньше операций чтения/записи

### Простое слияние (Simple Merge)

- Фиксированный размер серии на каждой итерации
- Простая логика реализации
- Контролируемое использование памяти

### Естественное слияние (Natural Merge)

- Размер серии определяется исходной отсортированностью данных
- Использует уже отсортированные последовательности (runs)
- Может быть более эффективно для частично отсортированных данных

## 🛠️ Технические детали

### Используемые возможности C++

- **Шаблоны** (Templates): Реализация сортировки как шаблонов для различных типов данных
- **Работа с файлами**: `<fstream>` для чтения/записи файлов
- **Файловая система**: `<filesystem>` для проверки существования файлов
- **Хронометраж**: `<chrono>` для точного измерения времени
- **Обработка исключений**: Try-catch для безопасной работы с файлами

### Требования памяти

- **Входной файл**: Хранит до N целых чисел
- **Временные файлы**: До 2N целых чисел для двухфазной сортировки
- **Буферы в памяти**: Минимальные буферы для операций чтения/записи

## 🐛 Обработка ошибок

Программа обрабатывает следующие ошибки:

- ❌ Файл не найден → Сообщение об ошибке и предложение создать новый файл
- ❌ Некорректный ввод → Повторное запрос корректного значения
- ❌ Ошибка при открытии файла → Перехват исключения и сообщение об ошибке
- ❌ Конец потока ввода (EOF) → Корректное завершение программы
