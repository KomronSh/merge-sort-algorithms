# Сравнение алгоритмов внешней сортировки слиянием

Курсовая работа по дисциплине "Структуры и алгоритмы обработки данных" (СиАКОД).

Проект представляет собой консольное приложение на C++, которое сортирует данные во внешних файлах разными вариантами сортировки слиянием, собирает статистику и позволяет запускать сравнительные тесты.

## Возможности

- создание входного файла `input.txt` со случайными числами или числами, введенными с клавиатуры;
- двухфазная сортировка простым слиянием;
- двухфазная сортировка естественным слиянием;
- однофазная сортировка простым слиянием;
- однофазная сортировка естественным слиянием;
- многопутевое слияние с задаваемым количеством путей `T`;
- автоматическое тестирование алгоритмов на наборах размером `100`, `500`, `1000`, `2000`, `5000`, `10000`;
- вывод количества сравнений, перемещений и времени выполнения;
- сохранение результатов тестирования в `benchmark_results.txt`.

## Использование

После запуска отображается главное меню:

```text
МЕНЮ
1. Создать новый входной файл
2. Выполнить двухфазную сортировку
3. Выполнить однофазную сортировку
4. Выполнить многопутевое слияние
5. Выполнить тестирование алгоритмов
6. Выход
Выберите действие:
```

Типичный порядок работы:

1. Выберите пункт `1` и создайте `input.txt`.
2. Укажите способ заполнения:
   - `1` - случайные числа;
   - `2` - ввод чисел с клавиатуры.
3. Укажите количество элементов.
4. Запустите нужный алгоритм через пункты `2`, `3` или `4`.
5. Посмотрите статистику в консоли.

При выборе многопутевого слияния программа дополнительно запрашивает количество путей `T`.

## Автоматическое тестирование

Пункт `5` главного меню запускает серию тестов для всех реализованных алгоритмов:

- двухфазная сортировка простым слиянием;
- двухфазная сортировка естественным слиянием;
- однофазная сортировка простым слиянием;
- однофазная сортировка естественным слиянием;
- многопутевое слияние.

Для каждого размера генерируется одинаковый набор случайных чисел, после чего каждый алгоритм сортирует свою копию данных. Результаты выводятся в консоль и записываются в файл `benchmark_results.txt` в табличном формате:

```text
Размер	Алгоритм	Сравнения	Перемещения	Время, мс	Время, нс
```

Для многопутевого слияния в автоматических тестах используется `T = 4`.

## Статистика

Каждый алгоритм заполняет структуру `SortStats`:

```cpp
struct SortStats {
    long long cmp_cnt = 0;
    long long move_cnt = 0;
    double duration_ns = 0;
};
```

Поля структуры:

- `cmp_cnt` - количество сравнений;
- `move_cnt` - количество перемещений или записей элементов;
- `duration_ns` - время выполнения в наносекундах.

В консоли время дополнительно выводится в миллисекундах.

## Краткое описание алгоритмов

### Двухфазная сортировка простым слиянием

Исходный файл разбивается на серии фиксированного размера, которые поочередно распределяются во временные файлы `temp1.txt` и `temp2.txt`. Затем серии сливаются обратно в основной файл. Размер серии на каждой итерации удваивается.

### Двухфазная сортировка естественным слиянием

Алгоритм ищет естественные возрастающие серии в исходных данных, распределяет их по файлам `f1.txt` и `f2.txt`, затем выполняет фазу слияния. Подход может быть эффективнее на частично отсортированных данных.

### Однофазная сортировка простым слиянием

Сначала данные распределяются по двум файлам, затем серии попеременно сливаются между парами файлов `f1.txt`, `f2.txt` и `f3.txt`, `f4.txt` до получения одной отсортированной серии.

### Однофазная сортировка естественным слиянием

Использует естественные серии и попеременное слияние между двумя парами временных файлов. После завершения итоговая серия копируется обратно в основной файл.

### Многопутевое слияние

Данные распределяются по `T` путям, после чего серии сливаются с использованием нескольких входных и выходных направлений. Количество путей задается пользователем перед запуском алгоритма.

## Файлы данных

Программа работает с текстовыми файлами, где числа записаны через пробел. Все имена файлов в коде заданы относительными путями, поэтому `input.txt`, временные файлы и `benchmark_results.txt` создаются в текущей рабочей директории запуска.

Основные файлы:

- `input.txt` - входной файл, который сортируется выбранным алгоритмом;
- `benchmark_input.txt` - временный файл для автоматических тестов, удаляется после завершения;
- `benchmark_results.txt` - результаты автоматического тестирования;
- `temp1.txt`, `temp2.txt`, `f1.txt`, `f2.txt`, `f3.txt`, `f4.txt` - временные файлы алгоритмов.

## Обработка ошибок

В программе предусмотрены проверки:

- корректности выбора пунктов меню;
- существования `input.txt` перед запуском сортировки;
- корректности количества элементов при создании файла;
- корректности ручного ввода чисел;
- ошибок открытия файлов;
- завершения ввода через EOF.

## Примечания

- Случайная генерация в ручном режиме создает числа в диапазоне от `0` до `3199`.
- Автоматические тесты генерируют числа в диапазоне от `100` до `10000`.
- Некоторые алгоритмы выводят дополнительную отладочную информацию для небольших файлов или отдельных фаз сортировки.
- Алгоритмы изменяют содержимое `input.txt`, поэтому перед сравнением разных методов лучше заново создать входной файл или использовать автоматическое тестирование.
